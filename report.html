Security Best Practices
SQL Injection
  • Use parameterized queries / prepared statements
  • Validate and whitelist all user inputs
  • Never expose raw SQL errors to users
  • Use least-privilege database accounts
  • Consider using an ORM (SQLAlchemy, Django ORM)
XSS
  • Encode all output — convert < > to HTML entities
  • Set Content-Security-Policy headers
  • Never use innerHTML or document.write() with user data
  • Set HttpOnly and Secure flags on cookies
  • Use frameworks that auto-escape output (React, Angular)
Authentication
  • Change all default credentials immediately
  • Add rate limiting and account lockout after 5 attempts
  • Set HttpOnly, Secure, SameSite flags on session cookies
  • Regenerate session ID after every login
  • Use HTTPS everywhere
Access Control
  • Implement Role-Based Access Control (RBAC)
  • Never use direct object IDs in URLs — use tokens
  • Always verify ownership server-side
  • Apply least privilege principle
  • Deny access by default — whitelist allowed actions
